Markup Practices

• Unless it is a major navigational element (e.g. logo, main navigation, sidebar) always use classes instead of ID's.

• Before creating a new style, we ask ourselves if we can extend an existing class instead.

• Always give abstract and high-level names to CSS classes. If an element requires customized styling, we extend the existing class to add greater specificity.

GOOD

.submit .button
.banner
.green .button
.box-form .login

BAD

.contactPageSubmitBtn
.aboutPageMainImage
.lightGreenButton
.loginScreen_v2

• Take advantage of advanced CSS selectors in order to target specific elements:
- Using p + p will target all <p> tags that are preceded by another <p> tag.
- Using .example > .code will target all elements .example that contain a child element of .code.
- li:last will apply to the last item in a list.

• Headline tags and anchor links have <span> tags enclosed within them, to give us more control over how their styles behave.

Buttons

Every "button" on the site inherits its properties from .button, adding more specific properties as needed. Once the styles are created, we can produce a wide variety of different-looking buttons without ever touching the CSS again.

The structure is as follows:

<a href="..." class="..."><span> Link Text </span></a>

The <a> tag provides the coloration and border effect for each button. The <span> tag contains the actual text, and provides padding around the text.
